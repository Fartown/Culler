## 问题定位
- 网格项容器使用 .clipped，导致靠边的覆盖图标（星级、旗标、颜色点）被裁切，见 [PhotoGridView.swift:L65-L123](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/PhotoGridView.swift#L65-L123) 与图片内部的裁剪 [PhotoGridView.swift:L153-L159](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/PhotoGridView.swift#L153-L159)。
- 右侧信息面板在 HSplitView 中宽度上限较低（maxWidth: 450），窗口收缩时被压缩；滚动内容与内边距可能造成视觉上的“显示不全”，见 [ContentView.swift:L106-L122](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L106-L122) 与 [InfoPanelView.swift:L7-L13](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/InfoPanelView.swift#L7-L13)。

## 修改方案：网格缩略图
- 移除缩略图项 ZStack 上的 .clipped，仅保留图片上的 .clipped，避免覆盖层被裁切。
- 将覆盖层改为 overlay(alignment: …) + padding/offset，避免使用 .position，提升 .zIndex 以免被图片遮挡。
- 统一覆盖图标与圆角的安全边距，确保在最小尺寸下也不触边。

示例修改（关键片段）：

```swift
ZStack(alignment: .topLeading) {
    AsyncThumbnailView(photo: photo, size: size)
        .cornerRadius(4)
        .overlay(
            RoundedRectangle(cornerRadius: 4)
                .strokeBorder(isSelected ? Color.accentColor : Color.clear, lineWidth: 3)
        )

    VStack(alignment: .leading, spacing: 4) { /* 旗标、星级 */ }
        .padding(.top, 6)
        .padding(.leading, 6)
        .zIndex(1)
}
.frame(width: size, height: size)
// 移除 .clipped()

// 颜色点改为顶部右侧对齐
.overlay(
    Circle()
        .fill(Color(photo.colorLabel.color))
        .frame(width: 12, height: 12)
        .padding(.top, 6)
        .padding(.trailing, 6),
    alignment: .topTrailing
)
```

图片内部保持：

```swift
Image(nsImage: image)
    .resizable()
    .aspectRatio(contentMode: .fill)
    .frame(maxWidth: .infinity, maxHeight: .infinity)
    .clipped()
```
