## 功能概述
- 在单图预览中增加左/右旋转 90° 功能，仅影响当前预览，不修改原文件。
- 旋转与现有缩放/拖拽/双击缩放交互兼容；旋转后重置偏移，避免图像移出视窗。

## UI改动
- SinglePhotoView 顶部覆盖条新增两个按钮：左旋与右旋，放置在计数文本右侧。
- 顶部 ToolbarView 的缩放按钮旁新增左旋/右旋按钮，提供全局入口。
- 图标优先使用 SF Symbols：rotate.left / rotate.right；若环境不支持则回退为 arrow.counterclockwise / arrow.clockwise。

## 快捷键与菜单
- 在 Commands 的 View 菜单中新增：
  - Rotate Left（⌘[）
  - Rotate Right（⌘]）
- 这两个命令向全局广播对应通知，与缩放命令同模式。

## 事件系统
- 在 Notification.Name 扩展中新增：rotateLeft / rotateRight。
- ToolbarView 旋转按钮与 Commands 菜单发出上述通知。
- SinglePhotoView 通过 .onReceive 监听通知并更新旋转状态。

## 渲染实现
- SinglePhotoView 增加状态：
  - @State private var rotationDegrees: Int = 0
- 在当前图片与 previousImage 的展示处统一应用旋转：

```swift
Image(nsImage: image)
  .resizable()
  .aspectRatio(contentMode: .fit)
  .rotationEffect(.degrees(Double(rotationDegrees)))
  .scaleEffect(scale)
  .offset(offset)
```
- 在 .onReceive rotateLeft/rotateRight 中：
  - withAnimation { rotationDegrees = (rotationDegrees ± 90 + 360) % 360 }
  - offset = .zero（旋转后居中）

## 状态重置
- 切换照片时将 rotationDegrees 重置为 0：
  - 在 .onChange(of: photo.id) 或 loadFullImage(crossfade: false) 分支中加入复位逻辑，以与现有 scale/baseScale/offset 重置一致。

## 动画与体验
- 对旋转操作使用 .easeInOut 动画，时长 0.2s，与现有淡入淡出一致。
- 旋转后将 offset 归零；保留当前缩放倍率，避免破坏用户缩放状态。

## 测试验证
- 打开单图预览，点击左/右旋按钮，图像应按 90° 步进旋转。
- 使用快捷键 ⌘[ / ⌘] 触发旋转；双击缩放与捏合缩放仍正常。
- 在滚轮缩放后再旋转，图像居中不偏移；切换至下一张，旋转状态归零。
- 跨帧渐变场景下（切换图片）前/后图均按相同角度渲染，避免视觉跳变。

## 兼容性与风险
- 若运行环境不支持 rotate.* 图标，自动使用箭头回退图标；不影响功能。
- SwiftUI rotationEffect 为视图级变换，不影响底层像素与保存；符合“仅预览旋转”需求。
- 旋转 90° 时可出现更明显的留白，外层已 .clipped，视图不溢出。
