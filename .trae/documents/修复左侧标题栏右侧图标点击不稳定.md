## 问题研判
- 截图显示“相册”标题右侧同时存在“＋”与一个向下的 chevron。代码中仅定义了“＋”按钮，[SidebarView.swift:L265-L278](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/SidebarView.swift#L265-L278)，chevron 很可能是 macOS 在 .listStyle(.sidebar) 下为可折叠 Section 提供的系统控件。
- 现状导致点击不稳的主要因素：
  1) “＋”按钮命中区域过小，仅有 16 宽，几乎等同图标本身；
  2) 系统 chevron 与“＋”贴得较近，且无额外右侧间距，容易误触；
  3) 右侧控件统一为 .plain 样式，hover 无反馈、低对比度，难以发现并精准点击；
  4) “文件夹”区的 Toggle 使用 .scaleEffect(0.7)，缩小了可点击面积。

## 修改要点
- 统一右侧控件的“最小热区+不改变几何”的交互规则：固定 24×24 点击区、contentShape(Rectangle)、轻度背景/边框/阴影作为 hover 反馈。
- 与系统 chevron 拉开间距：给各 Section header HStack 增加 .padding(.trailing, 10-12)。
- 移除“文件夹”Toggle 的缩放，改用 .controlSize(.small)。

## 示例代码片段
- 相册 Section header：

```swift
HStack {
    Text("相册").font(UIStyle.sectionHeaderFont).foregroundColor(.secondary)
    Spacer()
    Button(action: { showAlbumManager = true }) {
        Image(systemName: "plus")
            .font(.system(size: 13, weight: .semibold))
            .frame(width: 24, height: 24)
            .contentShape(Rectangle())
            .background(Color.primary.opacity(0.06))
            .clipShape(RoundedRectangle(cornerRadius: 6))
            .overlay(RoundedRectangle(cornerRadius: 6).stroke(Color.primary.opacity(0.12), lineWidth: 1))
    }
    .buttonStyle(.plain)
    .foregroundColor(.secondary)
}
.padding(.trailing, 12)
```

- 文件夹 Section header：

```swift
HStack {
    Text("文件夹").font(UIStyle.sectionHeaderFont).foregroundColor(.secondary)
    Spacer()
    Toggle("子文件夹", isOn: $includeSubfolders)
        .toggleStyle(.switch)
        .labelsHidden()
        .controlSize(.small)
}
.padding(.trailing, 12)
```

## 验证
- 进入应用，鼠标移入移出右侧图标区域：命中区域稳定、无抖动；
- 依次点击“相册”右侧“＋”、系统 chevron：相互不干扰，误触明显减少；
- 暗色背景下 hover 反馈对比度提升但不改变尺寸，点击更容易。

## 风险与回滚
- 样式增强会略微提升视觉存在感，如不满意可仅保留固定尺寸与 contentShape；
- 与系统 chevron 的距离通过 .padding(.trailing) 控制，数值可微调；
- 改动局部、易回滚，不影响数据与业务逻辑。
