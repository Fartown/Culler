## 变更目标
- 双击标题不再进入全屏，而是将窗口尺寸调整为当前屏幕的可用区域最大（不遮挡菜单栏与 Dock）。
- 行为发生在应用自定义工具栏的标题文本上，避免与系统标题栏的双击行为冲突。

- 标题文本位置（建议把双击手势移动至此）：[ToolbarView.swift:L27-L31](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/ToolbarView.swift#L27-L31)
- WindowAccessor（用于访问 NSWindow）：[WindowAccessor.swift](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/WindowAccessor.swift)

## 具体改动
- 将 .onTapGesture(count: 2) 从外层工具栏容器移到标题 Text 上，使“仅双击标题”触发。
- 替换当前动作（window.toggleFullScreen(nil)）为“最大化到屏幕可用区域”：

```swift
if let window = NSApp.keyWindow {
    let screen = window.screen ?? NSScreen.main
    if let visible = screen?.visibleFrame {
        window.setFrame(visible, display: true, animate: true)
    }
}
```

- 仍可使用现有 WindowAccessor 保存/获取 window；如不再需要其它窗口操作，可简化为直接使用 NSApp.keyWindow。
- 可选：若需要“再次双击恢复原尺寸”，增加状态记录上一次窗口 frame，并在下一次双击时还原。

## 兼容性与注意事项
- 多显示器：使用 window.screen?.visibleFrame，确保按窗口所在屏幕最大化。
- 不进入全屏：不再调用 toggleFullScreen，保持标准窗口模式（保留标题栏与菜单栏）。
- 与其他双击逻辑不冲突：单图视图与网格的双击用于内容缩放/打开，保持不变（参考 [SinglePhotoView.swift:L75-L84](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/SinglePhotoView.swift#L75-L84)、[PhotoGridView.swift:L42-L50](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/PhotoGridView.swift#L42-L50)）。

## 验证方式
- 运行应用，双击标题文本：窗口扩大到屏幕可用区域最大；不进入全屏。
- 再次双击：仍保持最大化（或按可选项恢复到之前尺寸）。
- 检查其他视图的双击行为不受影响。

## 回滚方案
- 如需恢复原行为，将双击动作改回 window.toggleFullScreen(nil)，或移除标题上的双击手势。
