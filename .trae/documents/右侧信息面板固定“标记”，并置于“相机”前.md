## 目标
- 在右侧信息面板中，“标记”分组固定常显（不再因无内容而隐藏），但保留折叠交互。
- 保持“标记”分组位于“相机”分组之前。

## 现状
- “标记”使用 DisclosureGroup，受 hasMarkingInfo 控制；当内容为空时整块不显示，并可折叠。
- 相关代码位于 [InfoPanelView.swift](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/InfoPanelView.swift#L70-L112)。

## 变更方案
1. 去除 hasMarkingInfo 对显示的条件控制：始终渲染“标记” DisclosureGroup。
2. 保留折叠交互：继续使用 `@State private var expandMarking` 作为折叠状态绑定，默认展开。
3. 空内容提示：当旗标/评分/颜色标签/标签均为空时，在 DisclosureGroup 内显示“暂无标记”说明，避免空白分组。
4. 排序保持：“标记”仍在“相机”之前，无需调整位置。

## 技术实现
- 文件：InfoPanelView.swift。
- 修改“标记”区块：
  - 删除 `if hasMarkingInfo { ... }` 条件，将 DisclosureGroup 直接常显。
  - 保留 `DisclosureGroup(isExpanded: $expandMarking)`，默认 `expandMarking = true`。
  - 在组内容中按已有逻辑渲染旗标/评分/颜色标签/标签；若全部为空，追加一行 `Text("暂无标记")`（样式为 caption/secondary，以与现有 UI 风格一致）。

- 示例（示意）：
```swift
// 始终显示“标记”DisclosureGroup，保留折叠
DisclosureGroup(isExpanded: $expandMarking) {
    VStack(alignment: .leading, spacing: UIStyle.groupSpacing) {
        if photo.flag != .none {
            KVRow(label: "旗标") { Image(systemName: flagIconName).font(.system(size: 16)).foregroundColor(flagIconColor) }
        }
        if displayedRating > 0 {
            KVRow(label: "评分") { /* 星级渲染同现有实现 */ }
        }
        if photo.colorLabel != .none {
            KVRow(label: "颜色标签") { Circle().fill(Color(photo.colorLabel.color)).frame(width: 12, height: 12) }
        }
        if let tags = photo.tags, !tags.isEmpty {
            KVRow(label: "标签") { Text(tags.map { $0.name }.joined(separator: ", ")).font(.system(size: 12)).lineLimit(1).truncationMode(.middle) }
        }
        if photo.flag == .none && displayedRating == 0 && photo.colorLabel == .none && (photo.tags?.isEmpty ?? true) {
            Text("暂无标记").font(.system(size: 12)).foregroundColor(.secondary)
        }
    }
} label: {
    Text("标记").font(.subheadline).foregroundColor(.secondary)
}
```

- “相机”分组位置保持不变（约 [InfoPanelView.swift:L114-L124](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/InfoPanelView.swift#L114-L124)）。
