## 改进目标
- 风格简洁、干净，和现有工具栏按钮保持一致
- 语义清晰，突出当前排序方式
- 去掉冗余装饰（多余边框、双图标），统一控件尺寸与对比度

## 现状与问题
- 现有下拉在 [ToolbarView.swift](file:///Users/zhangchao/workspace/dev/Culler/Culler/Culler/Views/ToolbarView.swift#L66-L103) 使用自定义 HStack 做 Menu 标签，额外叠加半透明深色背景与描边
- 同一行其它控件（“清除筛选”“同步”）采用系统 .bordered 小尺寸风格，视觉不统一
- 标签同时显示方向图标与自绘 chevron，信息重复，显得复杂

## 方案一（推荐）：使用系统“按钮式 Menu”风格
- 将 Menu 的标签替换为系统 Label，启用 .menuStyle(.button) + .controlSize(.small)，与旁边按钮统一
- 移除自定义背景/描边/附加 chevron，仅保留排序图标与当前标题
- 保留菜单项中的选中对勾，确保当前项清晰

代码改动（仅示意）：

```swift
Menu {
    ForEach(SortOption.allCases) { option in
        Button {
            sortOption = option
        } label: {
            option == sortOption ? Label(option.title, systemImage: "checkmark") : Text(option.title)
        }
    }
} label: {
    Label(sortOption.title, systemImage: "arrow.up.arrow.down")
        .font(.system(size: 12))
}
.menuStyle(.button)
.controlSize(.small)
```

效果
- 视觉更克制：与“清除筛选”“同步”一致的小尺寸边框按钮外观
- 信息更聚焦：仅显示排序图标与当前排序项，无重复 chevron
- 暗色主题下对比度恰当，不再出现额外描边与厚重背景

## 方案二：使用 Picker 的 Menu 风格
- 将 Menu 换为 Picker(selection: $sortOption) + .pickerStyle(.menu)
- 标签可用 Label("排序", systemImage: "arrow.up.arrow.down")，系统自动渲染弹出式外观

示意：

```swift
Picker(selection: $sortOption) {
    ForEach(SortOption.allCases) { Text($0.title).tag($0) }
} label: {
    Label("排序", systemImage: "arrow.up.arrow.down")
}
.pickerStyle(.menu)
.controlSize(.small)
```

效果
- 原生 macOS 弹出按钮样式，更加系统一致
- 但无法在标签中直接展示“当前项”文本（标签固定为“排序”），信息密度略低

## 细节优化
- 图标使用 .font(.system(size: 12))，颜色用 .secondary；文本用 .primary，字号统一为 12
- 间距：与右侧按钮保持 8pt 水平间距
- 仅保留一个图标（arrow.up.arrow.down），去除手动 chevron
