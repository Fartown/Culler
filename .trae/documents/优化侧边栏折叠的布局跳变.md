**问题诊断**
- 布局使用三列 HSplitView：[ContentView.swift](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L151-L156)，左/右侧边栏通过布尔条件直接移除视图：左侧在 [ContentView.swift:sidebarColumn](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L159-L187)，右侧在 [ContentView.swift:inspectorColumn](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L263-L289)。
- Toolbar 的折叠按钮切换 showLeftNav / showRightPanel：[ToolbarView.swift 左侧](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ToolbarView.swift#L18-L25) 与 [右侧](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ToolbarView.swift#L88-L96)。右栏标题也可折叠：[ContentView.swift](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L269-L276)。
- 由于在列级别“移除/添加”视图，HSplitView 会重算分配造成主区跳变。应改为“保持列存在，仅调整列宽”。

**优化方案**
- 始终保留三列：不再使用 if 条件隐藏列视图，左/右列固定挂载 SidebarView / InspectorView。
- 宽度折叠：为左右列引入宽度状态（leftWidth/rightWidth），折叠时将列宽动画为 0；展开时恢复到上次记忆值（例如 240/300）。
- 平滑动画：所有折叠/展开动作用 withAnimation(.easeInOut)，同时对宽度、透明度应用动画，避免布局跳变。
- 命中测试与可见性：折叠时 .opacity(0) + .allowsHitTesting(false)，避免不可见视图拦截事件。
- 列宽记忆：用 @SceneStorage 或 @AppStorage 保存 lastLeftWidth/lastRightWidth，保证用户体验一致。
- 主区域稳定：主列设置 .frame(maxWidth: .infinity) 并提高 layoutPriority，左右列宽变化时主区尽量不抖动。
- 右栏内容（InfoPanel）：确保内部使用固定最小宽度与滚动容器，避免在宽度变化时触发布局级联；参考 [InfoPanelView.swift](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/InfoPanelView.swift)。

**具体修改点**
- ContentView.swift：
  - 新增状态：leftCollapsed/rightCollapsed、leftWidth/rightWidth、lastLeftWidth/lastRightWidth；默认值例如 240/300。
  - 将 sidebarColumn/inspectorColumn 改为始终渲染对应视图，并设置 .frame(width: leftCollapsed ? 0 : leftWidth) 与 .frame(width: rightCollapsed ? 0 : rightWidth)，叠加 .opacity(leftCollapsed ? 0 : 1) 和 .allowsHitTesting(!leftCollapsed)。
  - 为 HSplitView 外层或列视图添加 .animation(.easeInOut, value: leftCollapsed/rightCollapsed)；折叠按钮处使用 withAnimation 包裹状态切换。
  - 通知处理 [.toggleLeftPanel/.toggleRightPanel](file:///Users/bytedance/workspace/ai-demo/culler/Culler/Culler/Views/ContentView.swift#L377-L387)：改为切换 collapsed 状态，并在展开时恢复 lastWidth。
- ToolbarView.swift：
  - 左右折叠按钮点击事件改为 withAnimation 切换 collapsed 状态，同时在展开时从持久化宽度恢复。
- InspectorView / InfoPanelView.swift：
  - 面板内容容器设置合理的 minWidth（如 ≥ 280），内部使用 ScrollView，避免在右栏宽度渐变时出现子视图重排导致的抖动；必要时对部分 DisclosureGroup 添加 .animation(value:) 保持一致性。
